# Portfolio, not master design

No single intervention is sufficient. Each useful intervention is worth doing in isolation, and none closes the whole loop alone, because the failure mode is information corruption across multiple lossy transmissions and no single intervention closes more than one. A portfolio of small interventions preserves system slack better than one big intervention trying to handle everything.

A single intervention treated as the answer for the whole becomes load-bearing for concerns it was never designed for and consumes the tolerances the rest of the system needs to absorb imperfection. Recognize the lighter form of this anti-pattern in the work that follows from these documents.
