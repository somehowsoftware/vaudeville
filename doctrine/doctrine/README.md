# Doctrine

The framing texts primed into every agent before it works — the law the agents run, and reference for the operator. The primary body is operative: executed at the action point, and authoritative wherever anything conflicts; the rest is read rather than executed ([vocabulary.md#doctrine](vocabulary.md#doctrine)).

Primary:

- [bearing/](bearing/) — how the agent carries itself in the work.
- [code/](code/) — how the artefact is built, structured, and verified.
- [practice/](practice/) — the methodology for coordinating ephemeral agents on one interdependent system.
- [vocabulary.md](vocabulary.md) — the coined terms, each defined once in relation to the others.

Secondary:

- [recipes/](recipes/) — worked procedures an agent reads and runs by hand.
- [quick-start.md](quick-start.md) — install, configure, and run daily work.
- [host-facts.md](host-facts.md) — how an operator's per-install facts reach an agent.
