# vaudeville-coordinator

vaudeville-coordinator contributes Assignment authoring and Assignment-state transitions to the Vaudeville scaffold.

Assignment authoring is the genus act of writing a new Assignment into the project tracker: composing its description, setting its Route, and wiring Depend edges to peer Assignments. The same act spans the four kinds (`/premise` writes a Premise, `/direction` a Direction, `/command` a Command, `/manual` a Manual, with `/tangent` writing a provisional Premise) through one create-plus-depend primitive. Each kind files itself with its own Type, admitted only on a Route its kind allows. `/premise` carries the discrimination gate that sorts the four, since a Premise and a Direction are stored alike but for a type label and nothing downstream catches a mislabel. When the Assignment is a Premise, its description takes the humble-snapshot shape; that frame is the Premise kind's, not the genus act's. Authoring takes two shapes. Most often an agent composes the description directly. For a tangent (a side-concern the operator captures without working it out) `vv tangent` instead builds a provisional body deterministically from a fixed set of captured fields, with no composition step, and classifies which Component the concern belongs to from vaudeville-core's per-Component descriptions rather than letting the capturing agent anchor on its own thread's Component.

A Direction is the Assignment whose ends are settled on a single named assumption (the hinge) while the means stay open: the body states the call, why it was made, the hinge it rests on, and what the call reopens to if the hinge gives. An agent reasons its own way to a Direction as it does a Premise, so it authors one on its own initiative. It is filed with Type Direction on the kind's one legal Route, check-in, through the same create-plus-depend primitive Premise authoring uses; like a Premise and unlike a Command, it can chain a spawn onto the filed id.

A Command is the operator's settled directive: ordered steps and authorization gates, executed rather than pressed, the inverse of a Premise's humble proposition. Because settled authority originates with the operator, an agent files a Command only on the operator's explicit invocation, and captures what the operator authorized without inventing detail. It is filed with Type Command and a Route of check-in or direct, through the same create-plus-depend primitive Assignment authoring uses, and it never chains a spawn: it carries the operator's authority, so it waits outside the pickup pool until the operator signs it off, the go-ahead that admits it for spawn.

A Manual delegates no discretion: the operator keeps the wheel and releases intent live, turn by turn, rather than deciding it all before the work begins. Like a Command it is operator-originated (an agent files one only on the operator's explicit invocation and captures the deferred instructions without inventing detail) and, also like a Command, it never chains a spawn and waits behind the operator's sign-off before it can be spawned and driven. It is filed with Type Manual and no Route, since a Manual classifies no conversation in advance; its intent arrives turn by turn.

Decomposition authors Assignments in sets, and in rounds rather than all at once: it cuts a settled plan-route design into the children that are implications of the meaning settled so far, pressure-tests each candidate's boundary against the Bob who would pick it up before any is filed, mints each as the kind its discretion calls for (a Direction for a settled leaf, a Premise where the ends stay open) and, when work remains beyond the round, files a continuation that re-decomposes the rest against what the round produced rather than against an up-front forecast. It authors the set through the same primitive single-Assignment authoring uses.

An Assignment-state transition moves an Assignment through its lifecycle without tearing down any worktree: claiming a Submitted or Returned Assignment, resolving it as delivered or abandoned, and returning it to the pickup pool. The transition writes only the tracker; a skill composes it with worktree teardown rather than coupling the two.

Coordinator also exposes the read primitives skills use to navigate the existing graph: fetch an Assignment, append commentary, and list what a resolved Assignment has unblocked: the Assignments it freed for autonomous pickup and, on a separate surface, the Command and Manual peers it freed that await the operator's sign-off.

vaudeville-coordinator does not spawn Bobs, tear down their worktrees, render a Command's first turn to the agent that runs it, or integrate the scaffold; those concerns live in vaudeville-bobiverse, vaudeville-cue, and vaudeville-ringmaster respectively.

vaudeville-coordinator-internal vocabulary is in [vocabulary.md](vocabulary.md). The cross-context Ubiquitous Language (Bob, Assignment, Route) is in `~/.vaudeville/doctrine/vocabulary.md`. The humble-snapshot shape an authored Premise takes is doctrine in `~/.vaudeville/doctrine/practice/premise-frame.md`.

## Collaborators

Two named boundaries.

**vaudeville-core** owns the anticorruption layer over the work tracker. Its name is a misnomer: so-called "core" is not a shared kernel but Vaudeville's outermost boundary, the layer that translates external systems (the work tracker, git hosting, the host config) into Vaudeville primitives. Reads and writes (fetching Assignments, transitioning their (State, Workflow) custom fields through `PROFILES` and `apply_bookkeeping`, attaching link edges, posting comments) go through "core"'s tracker module, its issue accessors, and its lifecycle-profile constants. vaudeville-coordinator does not name the tracker by product; the tracker product is "core"'s concern, hidden behind its anticorruption boundary. The per-Component register a tangent classifies a captured concern against also comes from "core", via `repo_descriptions()`: the published map of Component prefix to one-sentence summary.

**vaudeville-bobiverse** (via the federated `vv` facade) carries the spawn hand-off when an authoring skill's `--spawn` chains a Bob launch onto the filed Premise or Direction. The four spawn primitives (`vv spawn-preflight`, `vv spawn-launcher`, `vv spawn-target-repo`, `vv spawn-downstream`) plus `vv current-project` and `vv project-id` are bobiverse-owned; vaudeville-coordinator calls them through the deployed facade rather than reaching for a Python import. The edge looks bidirectional with bobiverse but is not a cycle: each Contributor calls the *deployed* facade `vv`, which routes to a Mirror of the other Contributor's published version.
